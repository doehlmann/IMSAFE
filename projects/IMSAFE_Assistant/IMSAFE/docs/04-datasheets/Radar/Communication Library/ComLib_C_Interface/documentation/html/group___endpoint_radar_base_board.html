typedef void(* consumption_def_cnf_callback_t) (consumption_def_cnf_ctx_t *p_context, int32_t protocol_handle, uint8_t endpoint, consumption_def_host_t *p_data)

This is the callback type executed on a parameter value confirmation event, triggered, by a parameter value request command.

  • Whenever a callback is issued, the following parameters are passed to the callback:
Parameters
[in]contextThe context data pointer, provided along with the callback itself through ep_radar_industrial_set_callback_bgt_lna_status.
[in]protocol_handleThe handle of the connection, the sending device is connected to.
[in]endpointThe number of the endpoint that has sent the message.
[in]p_datapointer to static memory of type consumption_def_host_t, holding the consumption values.

◆ ep_radar_base_board_is_compatible_endpoint()

int32_t ep_radar_base_board_is_compatible_endpoint ( int32_t  protocol_handle,
uint8_t  endpoint 
)

This function checks if an endpoint in a device is a Radar Baseboard endpoint.

This function checks type and version of the specified endpoint in a connected device and returns a code that indicates if that endpoint is compatible to the radar endpoint implementation in this module.

Parameters
[in]protocol_handleA handle to the connection to the sensor device.
[in]endpointThe endpoint in the connected device to be checked for compatibility.
Returns
If the specified endpoint is compatible to this implementation the function returns 0. If the endpoint is not compatible, a negative error code is returned.
